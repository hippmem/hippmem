# HIPPMEM 中文 NLP 能力说明

HIPPMEM 是一个**原生支持中文**的关联记忆引擎。与许多仅以英文为中心的向量数据库或记忆方案不同，HIPPMEM 从设计之初就将中文作为一等语言支持。本文档详细介绍 HIPPMEM 对中文的处理能力。

---

## 1. 为什么需要专门的中文支持

中文与英文在自然语言处理上存在根本性差异：

| 特性 | 英文 | 中文 |
|------|------|------|
| 分词 | 空格天然分隔 | 需要分词器（如 jieba） |
| 实体识别 | 大写字母提示 | 无视觉提示，依赖上下文 |
| 停用词 | "the", "a", "is" | "的", "了", "是" |
| BM25 检索 | 词干提取（stemming） | 分词 + N-gram |
| 情感分析 | 否定词 + 形容词 | 副词 + 形容词 + 语气词 |
| 因果关系 | "because", "therefore" | "因为…所以"、"导致"、"源于" |

因此，简单地复用英文 NLP pipeline 处理中文，会在分词和语义理解层面产生严重偏差。

---

## 2. HIPPMEM 的中文处理架构

HIPPMEM 通过 **locale 参数化架构** 实现对中文的原生支持。核心设计原则是：**算法与语言解耦**。

```
┌─────────────────────────────────────┐
│  算法层（src/，语言无关）              │
│  retrieve_api / enrich / extract     │
│  不含任何语言特定的硬编码字符串         │
└────────────┬────────────────────────┘
             │ 遍历 active_locales()
             ▼
┌─────────────────────────────────────┐
│  LangData 层（lang/<locale>.rs）     │
│  zh.rs  ← 所有中文 NLP 模式集中于此   │
│  en.rs  ← 英文 NLP 模式             │
└─────────────────────────────────────┘
```

### 2.1 中文分词

使用 **jieba** 分词器（Rust 绑定），支持：

- 精确模式（默认）：`"我爱自然语言处理"` → `["我", "爱", "自然语言", "处理"]`
- 搜索引擎模式：对长词再切分，提高召回率
- 自定义词典：支持添加领域术语

分词结果是 BM25 全文检索和实体提取的基础。

### 2.2 实体提取

HIPPMEM 的实体提取器在中文模式下使用 jieba 分词 + TF-IDF 关键词提取，识别：

- 人名：`"陈明"`、`"李华"`
- 地名：`"杭州"`、`"西湖区"`
- 组织/项目名：通过大写/引号/书名号识别
- 技术术语：基于词频和上下文

与英文不同，中文实体提取不依赖大写字母提示，而是依赖：
1. jieba 的词性标注（POS tagging）
2. TF-IDF 显著性评分
3. 上下文模式匹配

### 2.3 BM25 中文全文检索

基于 Tantivy 的中文 BM25 索引，使用 jieba 作为 tokenizer：

- jieba 分词后建立倒排索引
- BM25 评分算法原生支持中文分词的统计特性
- 支持中文停用词过滤（"的"、"了"、"是"、"在"等）
- 与英文 BM25 共享同一索引结构，查询时按 locale 选择 tokenizer

### 2.4 中文停用词

HIPPMEM 内置了常用的中文停用词表，覆盖：

- 结构助词：的、地、得
- 语气词：了、呢、吧、吗、啊
- 介词：在、于、对、从
- 连词：和、与、或、而、但
- 代词：我、你、他、她、它、这、那

停用词在 BM25 索引和关键词提取阶段被过滤，提高信噪比。

### 2.5 中文情感分析

基于关键词的情感极性判断：

- 正面词：`"优秀"`、`"高效"`、`"喜欢"`、`"推荐"`
- 负面词：`"糟糕"`、`"缓慢"`、`"讨厌"`、`"放弃"`
- 否定词：`"不"`、`"没"`、`"无"`——翻转后续情感词极性
- 程度副词：`"非常"`、`"极其"`、`"稍微"`——调整情感强度

### 2.6 中文因果识别

基于因果连接词模式匹配：

- 原因 → 结果：`"因为…所以"`、`"由于…因此"`
- 结果 → 原因：`"…是因为"`、`"…源于"`
- 隐含因果：`"导致"`、`"造成"`、`"引发"`
- 决策因果：`"选择…因为"`、`"放弃…由于"`

## 3. 添加新语言

HIPPMEM 的 locale 参数化架构使得添加新语言非常简单，无需修改任何算法代码：

**第 1 步**：创建 `crates/hippmem-model/src/lang/<locale>.rs`，定义该语言的 NLP 模式（停用词、情感关键词、因果对等）

**第 2 步**：在 `active_locales()` 中注册一行

**第 3 步**：创建对应的测试 fixture：`tests/fixtures/<purpose>/<locale>.json`

算法文件和测试逻辑零改动。目前支持的 locale：

| Locale | 语言 | 支持程度 |
|--------|------|----------|
| `zh` | 中文 | 完整支持（分词、实体、情感、因果、BM25） |
| `en` | 英文 | 完整支持 |

---

## 4. 实际效果

HIPPMEM 在中文记忆场景下的核心能力：

- ✅ 中文记忆的写入、检索、关联发现——与英文同等完整
- ✅ 中文分词驱动的 BM25 全文检索——比纯 embedding 检索更精准
- ✅ 中文因果链追踪——"因为…所以…"自动建立因果边
- ✅ 中文实体网络——人名、地名、项目名的自动关联
- ✅ 中英文混合记忆——同一引擎同时处理中英文记忆，自动建立跨语言关联

## 5. 了解更多
